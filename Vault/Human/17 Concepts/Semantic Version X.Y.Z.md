# Semantic Version X.Y.Z

1. x = MAJOR version when you make incompatible API changes
	1. The first Major version (1.0.0) is the official release of the product

2. y = MINOR version when you add functionality in a backward compatible manner
	1. Prior to version 1.0.0, these can make incompatible API changes

3. z = PATCH version when you make backward compatible bug fixes
